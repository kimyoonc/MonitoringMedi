# ROADMAP: 장기 처방 의약품 복약 관리 시스템

**문서 버전:** 1.0

**기반 문서:** docs/PRD.md v1.0

## 1. 프로젝트 개요

### 1.1 제품 설명

약국이 장기 처방 의약품을 복용하는 환자를 대상으로 복약 기간 전반에 걸쳐 지속적인 관리 역할을 수행할 수 있도록 지원하는 복약 관리 시스템.

### 1.2 핵심 사용자

| 사용자 | 역할 |
|---|---|
| 약사 (Primary User) | 복약 관리 계획 수립, 조제 단계 관리, 복약 상태 기록, 이상 반응 모니터링 |
| 환자 (End User) | 방문 일정 확인, 이상 반응 신고, 의약품 보관 상태 신고 |

### 1.3 기술 스택

| 영역 | 선택 | 비고 |
|---|---|---|
| 플랫폼 | 반응형 웹 | 모바일(375px~) / 태블릿(768px) / PC(1280px) |
| 프론트엔드 | React + TypeScript (Vite) | SPA, 모바일 친화적 UI |
| 백엔드 | Express.js Mock 서버 | 시나리오별 고정 JSON 응답, 실 DB 없음 |
| 데이터 | 테스트 픽스처 (JSON) | 시나리오별 샘플 데이터 정의 |

### 1.4 핵심 기능 목록

| 기능 ID | 기능명 | 우선순위 |
|---|---|---|
| F-001 | 복약 관리 계획 수립 (방문 일정 자동 계산) | P0 (필수) |
| F-002 | 단계적 조제 관리 | P0 (필수) |
| F-003 | 환자 방문 일정 알림 (D-3, D-1) | P1 (중요) |
| F-004 | 복약 상태 기록 | P0 (필수) |
| F-005 | 의약품 교환/보충 관리 | P1 (중요) |
| F-006 | 약물 상호작용 확인 | P1 (중요) |
| F-007 | 복약 관리 현황 대시보드 | P1 (중요) |

## 3. 스프린트 상세 계획

### Sprint 1: 프로젝트 셋업 + 핵심 인프라

**목표:** 개발 환경 구성 및 프론트엔드/백엔드 기본 골격 구축. 이후 스프린트에서 기능 구현이 바로 시작될 수 있도록 인프라를 완비한다.

**상태:** ⬜ 미시작

#### 포함 작업

| 작업 | 설명 |
|---|---|
| S1-01 | Vite + React + TypeScript 프로젝트 초기화 (`frontend/`) |
| S1-02 | Express.js Mock 서버 초기화 (`backend/`) |
| S1-03 | 반응형 레이아웃 기반 설정 (375px / 768px / 1280px 브레이크포인트) |
| S1-04 | 라우팅 구조 설계 (약사 화면 / 환자 화면 분리) |
| S1-05 | Mock API 엔드포인트 뼈대 정의 (환자, 처방, 복약 관리 계획, 방문 기록, 교환 이력) |
| S1-06 | 테스트 픽스처 데이터 (JSON) 초기 정의 |
| S1-07 | 공통 컴포넌트 기반 구성 (헤더, 내비게이션, 레이아웃 컴포넌트) |
| S1-08 | 개발/빌드 스크립트 설정 (`npm run dev`, `npm run build`) |

#### 완료 조건

- ✅ `npm run dev` 실행 시 프론트엔드(React)와 백엔드(Express) 동시 기동
- ✅ 약사 화면과 환자 화면 진입점(라우트) 접근 가능
- ✅ Mock API 엔드포인트 호출 시 고정 JSON 응답 반환
- ✅ 모바일(375px), 태블릿(768px), PC(1280px) 기본 레이아웃 렌더링 이상 없음

### Sprint 2: P0 핵심 기능 구현

**목표:** 복약 관리 시스템의 핵심인 P0 기능 3종(F-001, F-002, F-004)을 완전히 구현하여 MVP(v1.0)를 달성한다.

**상태:** ⬜ 미시작

**의존성:** Sprint 1 완료 후 시작

#### 포함 기능

| 기능 ID | 기능명 | 우선순위 |
|---|---|---|
| F-001 | 복약 관리 계획 수립 (방문 일정 자동 계산) | P0 |
| F-002 | 단계적 조제 관리 | P0 |
| F-004 | 복약 상태 기록 | P0 |

#### F-001: 복약 관리 계획 수립

- 총 처방 기간, 처방 의약품 목록, 환자 희망 방문 횟수를 입력받아 조제 단위 및 방문 일정 자동 계산
- 단계별 조제 계획(날짜, 조제 수량, 방문 일정) 생성 및 표시
- Mock API: `POST /api/plans` — 계획 생성, `GET /api/plans/:id` — 계획 조회

**인수 기준:** 처방 기간과 방문 횟수 입력 시 조제 단위 및 일정이 자동 계산되어 화면에 표시됨

#### F-002: 단계적 조제 관리

- 전체 처방량을 방문 횟수에 따라 분할하여 단계별 조제 지원
- 각 단계 조제 시 이전 단계 복약 상태 확인 필수 (이전 방문 기록 없으면 다음 단계 조제 불가)
- Mock API: `GET /api/plans/:id/steps` — 조제 단계 조회, `POST /api/visits/:id/dispense` — 조제 처리

**인수 기준:** 이전 방문 기록 확인 후에만 다음 단계 조제 진행 가능

#### F-004: 복약 상태 기록

- 방문마다 환자의 복약 상태, 이상 반응, 상담 내용 기록
- 기록 항목: 복약 순응도, 부작용 여부, 의약품 보관 상태, 약사 메모
- 방문 이력 목록 조회 가능
- Mock API: `POST /api/visits` — 방문 기록 저장, `GET /api/patients/:id/visits` — 방문 이력 조회

**인수 기준:** 방문마다 복약 상태 전 항목 입력 및 저장, 이력 조회 가능

#### 테스트 케이스 (Sprint 2)

| TC | 시나리오 | 기대 결과 |
|---|---|---|
| TC-01 | 환자 신규 등록 및 복약 관리 계획 수립 | 방문 일정 및 단계별 조제 계획 생성 |
| TC-02 | 1차 방문 - 정상 조제 | 1단계 의약품 조제 및 방문 기록 저장 |
| TC-03 | 정기 방문 - 복약 상태 정상 | 다음 단계 조제 진행, 이력 누적 |

#### 완료 조건

- ✅ TC-01, TC-02, TC-03 Mock 시나리오 전체 동작 확인
- ✅ F-001: 처방 기간·방문 횟수 입력 → 자동 계산 일정 표시
- ✅ F-002: 이전 방문 기록 없을 시 다음 단계 조제 버튼 비활성화
- ✅ F-004: 복약 상태 전 항목 입력 저장 및 이력 목록 표시
- ✅ 모바일(375px) / PC(1280px) 주요 화면 레이아웃 이상 없음

### Sprint 3: P1 확장 기능 구현

#### 포함 기능

| 기능 ID | 기능명 | 우선순위 |

| F-006 | 약물 상호작용 확인 | P1 |

#### F-003: 환자 방문 일정 알림

- 예정된 방문일 D-3, D-1 시점에 환자에게 알림 발송 (Mock 처리)
- 환자 화면: 다가오는 방문 일정 알림 목록 표시
- Mock API: `GET /api/patients/:id/notifications` — 알림 목록 조회

**인수 기준:** D-3, D-1 시점에 환자에게 알림이 발송됨 (Mock 발송 처리)

#### F-005: 의약품 교환/보충 관리

- 오염·훼손·유통기한 문제 발생 시 의약품 교환 처리 및 이력 관리
- 교환 사유(오염, 훼손, 유통기한 임박), 교환 수량, 처리 방법 기록
- Mock API: `POST /api/exchanges` — 교환 처리, `GET /api/visits/:id/exchanges` — 교환 이력 조회

**인수 기준:** 교환 사유·수량 입력 후 교환 이력에 기록됨

#### F-006: 약물 상호작용 확인

- 신규 처방 의약품과 기존 복용 의약품 간 상호작용 경고 표시
- 경고 발생 시 약사 확인 후 진행하는 플로우
- Mock API: `POST /api/interactions/check` — 상호작용 확인 (고정 Mock 응답)

**인수 기준:** 상호작용 대상 의약품 추가 시 경고가 화면에 표시됨

#### F-007: 복약 관리 현황 대시보드

- 약사가 관리 중인 환자 전체 현황 한눈에 파악
- 표시 항목: 오늘 방문 예정 환자, 조제 대기 환자, 이상 반응 환자
- Mock API: `GET /api/dashboard` — 대시보드 현황 데이터

**인수 기준:** 오늘 방문 예정·조제 대기·이상 반응 환자 수가 정확히 집계됨

| TC-06 | 유통기한 임박 의약품 교환 | 교환 처리 및 이력 기록 |
