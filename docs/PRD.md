**문서 버전:** 3.0

**최종 수정일:** 2026-03-15
**상태:** 구현 완료 (Sprint 1~3 + DB 연동 + UI 개선 + 추가 기능)

## 1. 프로젝트 개요

### 1.1 제품 설명
약국이 장기처방 의약품 분할조제를 받는 환자를 대상으로, 복약 기간 전반에 걸쳐 지속적인 관리 역할을 수행할 수 있도록 지원하는 복약 관리 시스템 및 서비스 방법.

### 1.2 배포 현황

| 항목 | URL |
|---|---|
| 프론트엔드 | https://monitoringmedi.vercel.app |
| 백엔드 API | https://monitoringmedi.onrender.com |
| 소스코드 | https://github.com/kimyoonc/MonitoringMedi |

### 1.3 핵심 가치 제안
- **환자**: 안전하고 체계적인 복약 관리를 통해 부작용 및 오남용 위험 감소
- **약국/약사**: 단순 조제를 넘어 복약 관리 전문가로서 역할 강화 및 수익 기회 확대
- **의료 시스템**: 약국을 지역사회 만성질환 관리의 접점으로 활용

## 2. 문제 정의

### 2.1 현재 상황의 문제점
현대 의료 환경에서 만성질환 및 장기 치료가 필요한 환자에 대한 의약품 관리는 환자가 약국을 방문하는 시점에서 **일회성**으로만 이루어진다.

| 문제 유형 | 상세 내용 |
|---|---|
| 건강 상태 변화 미모니터링 | 복용 과정에서 발생하는 부작용·이상 반응에 대한 지속적 관리 부재 |
| 의약품 보관 문제 | 장기간 가정 보관 시 오염, 변질, 분실 발생 가능 |
| 유통기한 위험 | 남은 의약품을 유통기한 확인 없이 복용하는 위험 |
| 약물 상호작용 관리 부재 | 다른 의약품과의 중복 복용 또는 상호작용 미관리 |
| 병원의 관리 한계 | 진료 중심 특성상 일상적 복약 상태의 지속 관리 불가 |

### 2.2 왜 약국이 해결해야 하는가
- 환자의 **접근성이 높음** (병원 대비 방문 빈도 높음)
- **의약품 전문가**인 약사가 상주
- 지역사회 내 **1차 의료 접점**으로서 최적 위치

## 3. 목표

### 3.1 제품 목표
1. 약국 중심의 장기 처방 의약품 단계적 조제 및 복약 관리 체계 구축
2. 환자-약사 간 지속적 관계 형성을 통한 복약 순응도 향상
3. 의약품 오남용 및 부작용 사고 예방

### 3.2 비즈니스 목표
1. 약국의 전문 서비스 역량 강화로 차별화된 경쟁력 확보
2. 환자 방문 빈도 증가를 통한 부가 수익 창출
3. 고객 신뢰도 향상 및 신규 고객 유입 효과

## 4. 대상 사용자

### 4.1 주요 사용자

**약사 (Primary User)**
- 환자의 복약 관리 계획 수립 및 실행
- 방문 일정 관리 및 조제 단계 관리
- 복약 상태 기록 및 이상 반응 모니터링

**환자 (End User)**
- 장기 처방 의약품을 복용 중인 만성질환자
- 복약 일정 관리 및 방문 알림 수신
- 의약품 보관 상태 및 이상 반응 신고

### 4.2 사용자 시나리오

**시나리오: 장기 처방 환자의 단계적 복약 관리**

```
1. [초기 방문]
   - 환자가 장기 처방전을 가지고 약국 방문
   - 약사가 전체 처방 기간 및 환자 희망 방문 횟수 확인
   - 복약 관리 계획(방문 일정, 조제 단위) 설정
   - 1차 조제 의약품 지급 및 복약 지도

2. [복약 기간 중 이슈 방문 - 수시]
   - 보관 중 의약품 오염/훼손 발견 시 방문
   - 유통기한 임박 의약품 교환 요청
   - 복용 중 이상 반응/부작용 발생 시 상담
   - 다른 의약품과의 상호작용 우려 시 확인

3. [정기 방문 - 계획된 일정]
   - 약사가 환자 상태 확인 (건강 상태, 복약 순응도)
   - 이전 복약 기간 이슈 검토
   - 다음 단계 의약품 조제 및 지급
   - 업데이트된 복약 지도 제공
```

## 5. 핵심 기능 요구사항

### 5.1 복약 관리 계획 수립 (F-001) ✅ 구현 완료
- **기능**: 처방 기간과 환자 방문 희망 횟수를 기반으로 조제 단위 및 방문 일정 자동 계산
- **입력**: 총 처방 기간, 처방 의약품 목록, 환자 희망 방문 횟수
- **출력**: 단계별 조제 계획 (날짜, 조제 수량, 방문 일정)
- **계산 로직**: `조제단위 = floor(총처방일 / 방문횟수)`, 잔여일은 마지막 단계에 추가
- **우선순위**: P0 (필수)

### 5.2 단계적 조제 관리 (F-002) ✅ 구현 완료
- **기능**: 전체 처방량을 방문 횟수에 따라 분할하여 단계별 조제 지원
- **조건**: 각 단계 조제 시 이전 단계 복약 상태 확인 필수 (`canDispense` 검증)
- **오류 처리**: 이전 단계 미완료 시 `PREVIOUS_VISIT_REQUIRED` 응답
- **우선순위**: P0 (필수)

### 5.3 환자 방문 일정 알림 (F-003) ✅ 구현 완료
- **기능**: 예정된 방문일 전 환자에게 알림 표시
- **설정**: 방문 D-3 (info 배지), D-1 (warning 배지) 사전 알림
- **읽음 처리**: 알림 카드 클릭 시 읽음 상태 업데이트
- **우선순위**: P1 (중요)

### 5.4 복약 상태 기록 (F-004) ✅ 구현 완료
- **기능**: 방문마다 환자의 복약 상태, 이상 반응, 상담 내용 기록
- **항목**: 복약 순응도(양호/보통/불량), 이상반응 여부 및 내용, 의약품 보관 상태, 약사 메모
- **이력**: 환자 상세 페이지에서 방문 이력 타임라인 조회
- **우선순위**: P0 (필수)

### 5.5 의약품 교환/보충 관리 (F-005) ✅ 구현 완료
- **기능**: 오염·훼손·유통기한 문제 발생 시 의약품 교환 처리 및 이력 관리
- **교환 사유**: 오염/훼손, 유통기한 임박, 기타
- **처리**: 교환 사유, 교환 수량, 단위, 처리 방법, 약사 메모 기록
- **이력 조회**: 환자별 교환 이력 탭 제공
- **우선순위**: P1 (중요)

### 5.6 약물 상호작용 확인 (F-006) ✅ 구현 완료
- **기능**: 복약 관리 계획 수립 시 의약품 추가마다 상호작용 자동 체크
- **경고 수준**: high (빨간색 경고), moderate (노란색 경고)
- **UX**: debounce 500ms 적용, 실시간 인라인 경고 표시
- **데이터**: 상호작용 DB (4쌍 정의)
- **우선순위**: P1 (중요)

### 5.7 복약 관리 현황 대시보드 (F-007) ✅ 구현 완료
- **기능**: 약사가 관리 중인 환자 전체 현황 한눈에 파악
- **요약 카드**: 오늘 방문 예정, 조제 대기, 이상 반응 환자 수 (SVG 아이콘 + 그라디언트)
- **섹션**: 오늘 방문 예정 / 조제 대기 / 이상 반응 환자 목록 (섹션별 이모지 아이콘)
- **레이아웃**: PC 3컬럼, 태블릿 2컬럼, 모바일 1컬럼
- **우선순위**: P1 (중요)

### 5.8 복약 순응도 차트 (F-008) ✅ 구현 완료
- **기능**: 환자별 방문 차수별 복약 순응도 추이를 SVG 막대 차트로 시각화
- **표시**: 양호(초록) / 보통(주황) / 불량(빨강) 색상 코딩, 비율 요약 통계
- **위치**: 환자 상세 페이지 방문 이력 위
- **우선순위**: P2 (개선)

### 5.9 방문 캘린더 뷰 (F-009) ✅ 구현 완료
- **기능**: 전체 환자의 방문 일정을 월별 달력으로 시각화
- **표시**: 날짜별 방문 도트 (완료=초록 / 예정=파랑 / 대기=주황)
- **인터랙션**: 날짜 클릭 시 해당 날 방문 환자 목록 슬라이드 패널, 환자 상세 이동
- **API**: `GET /api/calendar?year=&month=` (PlanStep + Patient JOIN)
- **위치**: `/pharmacist/calendar` (네비게이션 📅 탭)
- **우선순위**: P2 (개선)

## 8. 성공 지표 (KPI)

### 8.1 환자 안전 지표
- 의약품 부작용 발생률 감소 (기준 대비 %)
- 유통기한 만료 의약품 복용 사고 건수: 0건 목표
- 복약 순응도 향상율

### 8.2 약국 운영 지표
- 장기 처방 환자 1인당 평균 방문 횟수 증가
- 복약 관리 서비스 등록 환자 수
- 고객 만족도 (NPS)

### 8.3 비즈니스 지표
- 복약 관리 관련 부가 서비스 매출
- 신규 고객 유입율 (관리 환자 소개 통한)
- 환자 재방문율

## 9. 제약 사항 및 가정

### 9.1 제약 사항
- 기존 약국 처방 조제 시스템(DUR 등)과의 연동 필요 (현재 미연동)
- 의약품 분할 조제에 관한 법적 요건 검토 필요
- 약사 1인당 관리 가능한 환자 수 현실적 한계 고려
- Render 무료 플랜: 15분 비활성 시 절전 모드 (첫 요청 30~60초 지연)

### 9.2 가정
- 장기 처방(30일 이상) 환자를 주 대상으로 함
- 환자가 스마트폰 사용 가능 (알림 기능 활용)
- 약국이 기본적인 IT 인프라(인터넷, PC 등) 보유
