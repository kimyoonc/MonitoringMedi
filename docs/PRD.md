# PRD: 장기 처방 의약품 복약 관리 시스템

**문서 버전:** 1.0
**작성일:** 2026-03-14
**상태:** 초안

---

## 1. 프로젝트 개요

### 1.1 제품 설명
약국이 장기 처방 의약품을 복용하는 환자를 대상으로 복약 기간 전반에 걸쳐 지속적인 관리 역할을 수행할 수 있도록 지원하는 복약 관리 시스템 및 서비스 방법.

### 1.2 핵심 가치 제안
- **환자**: 안전하고 체계적인 복약 관리를 통해 부작용 및 오남용 위험 감소
- **약국/약사**: 단순 조제를 넘어 복약 관리 전문가로서 역할 강화 및 수익 기회 확대
- **의료 시스템**: 약국을 지역사회 만성질환 관리의 접점으로 활용

---

## 2. 문제 정의

### 2.1 현재 상황의 문제점
현대 의료 환경에서 만성질환 및 장기 치료가 필요한 환자에 대한 의약품 관리는 환자가 약국을 방문하는 시점에서 **일회성**으로만 이루어진다.

| 문제 유형 | 상세 내용 |
|---|---|
| 건강 상태 변화 미모니터링 | 복용 과정에서 발생하는 부작용·이상 반응에 대한 지속적 관리 부재 |
| 의약품 보관 문제 | 장기간 가정 보관 시 오염, 변질, 분실 발생 가능 |
| 유통기한 위험 | 남은 의약품을 유통기한 확인 없이 복용하는 위험 |
| 약물 상호작용 관리 부재 | 다른 의약품과의 중복 복용 또는 상호작용 미관리 |
| 병원의 관리 한계 | 진료 중심 특성상 일상적 복약 상태의 지속 관리 불가 |

### 2.2 왜 약국이 해결해야 하는가
- 환자의 **접근성이 높음** (병원 대비 방문 빈도 높음)
- **의약품 전문가**인 약사가 상주
- 지역사회 내 **1차 의료 접점**으로서 최적 위치

---

## 3. 목표

### 3.1 제품 목표
1. 약국 중심의 장기 처방 의약품 단계적 조제 및 복약 관리 체계 구축
2. 환자-약사 간 지속적 관계 형성을 통한 복약 순응도 향상
3. 의약품 오남용 및 부작용 사고 예방

### 3.2 비즈니스 목표
1. 약국의 전문 서비스 역량 강화로 차별화된 경쟁력 확보
2. 환자 방문 빈도 증가를 통한 부가 수익 창출
3. 고객 신뢰도 향상 및 신규 고객 유입 효과

---

## 4. 대상 사용자

### 4.1 주요 사용자

**약사 (Primary User)**
- 환자의 복약 관리 계획 수립 및 실행
- 방문 일정 관리 및 조제 단계 관리
- 복약 상태 기록 및 이상 반응 모니터링

**환자 (End User)**
- 장기 처방 의약품을 복용 중인 만성질환자
- 복약 일정 관리 및 방문 알림 수신
- 의약품 보관 상태 및 이상 반응 신고

### 4.2 사용자 시나리오

**시나리오: 장기 처방 환자의 단계적 복약 관리**

```
1. [초기 방문]
   - 환자가 장기 처방전을 가지고 약국 방문
   - 약사가 전체 처방 기간 및 환자 희망 방문 횟수 확인
   - 복약 관리 계획(방문 일정, 조제 단위) 설정
   - 1차 조제 의약품 지급 및 복약 지도

2. [복약 기간 중 이슈 방문 - 수시]
   - 보관 중 의약품 오염/훼손 발견 시 방문
   - 유통기한 임박 의약품 교환 요청
   - 복용 중 이상 반응/부작용 발생 시 상담
   - 다른 의약품과의 상호작용 우려 시 확인

3. [정기 방문 - 계획된 일정]
   - 약사가 환자 상태 확인 (건강 상태, 복약 순응도)
   - 이전 복약 기간 이슈 검토
   - 다음 단계 의약품 조제 및 지급
   - 업데이트된 복약 지도 제공
```

## 5. 핵심 기능 요구사항

### 5.1 복약 관리 계획 수립 (F-001)
- **기능**: 처방 기간과 환자 방문 희망 횟수를 기반으로 조제 단위 및 방문 일정 자동 계산
- **입력**: 총 처방 기간, 처방 의약품 목록, 환자 희망 방문 횟수
- **출력**: 단계별 조제 계획 (날짜, 조제 수량, 방문 일정)
- **우선순위**: P0 (필수)

### 5.2 단계적 조제 관리 (F-002)
- **기능**: 전체 처방량을 방문 횟수에 따라 분할하여 단계별 조제 지원
- **조건**: 각 단계 조제 시 이전 단계 복약 상태 확인 필수
- **우선순위**: P0 (필수)

### 5.3 환자 방문 일정 알림 (F-003)
- **기능**: 예정된 방문일 전 환자에게 알림 발송 (앱 푸시, SMS 등)
- **설정**: 방문 D-3, D-1 사전 알림
- **우선순위**: P1 (중요)

### 5.4 복약 상태 기록 (F-004)
- **기능**: 방문마다 환자의 복약 상태, 이상 반응, 상담 내용 기록
- **항목**: 복약 순응도, 부작용 여부, 의약품 보관 상태, 약사 메모
- **우선순위**: P0 (필수)

### 5.5 의약품 교환/보충 관리 (F-005)
- **기능**: 오염·훼손·유통기한 문제 발생 시 의약품 교환 처리 및 이력 관리
- **처리**: 교환 사유, 교환 수량, 처리 방법 기록
- **우선순위**: P1 (중요)

### 5.6 약물 상호작용 확인 (F-006)
- **기능**: 신규 처방 의약품과 기존 복용 의약품 간 상호작용 경고
- **연동**: 약물 상호작용 DB 연동
- **우선순위**: P1 (중요)

### 5.7 복약 관리 현황 대시보드 (F-007)
- **기능**: 약사가 관리 중인 환자 전체 현황 한눈에 파악
- **항목**: 오늘 방문 예정 환자, 조제 대기 환자, 이상 반응 환자
- **우선순위**: P1 (중요)

## 6. 기술 구성

### 6.1 기술 스택

| 영역 | 선택 | 비고 |
|---|---|---|
| 플랫폼 | 반응형 웹 (Responsive Web) | 모바일(375px~) / 태블릿(768px) / PC(1280px) |
| 프론트엔드 | React + TypeScript | Vite 기반 SPA, 모바일 친화적 UI |
| 백엔드 | Express.js Mock 서버 | 시나리오별 고정 JSON 응답 반환, 실 DB 없음 |
| 데이터 | 테스트 픽스처 (JSON) | 시나리오별 샘플 데이터 정의 |

### 6.2 백엔드 구성 방침

- 모든 API는 **테스트 케이스(Mock 데이터)** 기반으로 동작
- 실제 DB 연동 없이 시나리오별 고정 응답 반환
- 주요 시나리오별 테스트 케이스 사전 정의:
  - 정상 방문 및 조제
  - 이상 반응 신고
  - 의약품 교환 요청
  - 약물 상호작용 경고

### 6.3 주요 구성 요소

```
┌──────────────────────────────────────────────────┐
│         반응형 웹 (모바일/PC 공통)                  │
│                                                   │
│  ┌──────────────┐    ┌──────────────────────┐    │
│  │  약사 화면    │    │     환자 화면         │    │
│  │ (관리/조제)  │    │  (일정 확인/신고)     │    │
│  └──────┬───────┘    └──────────┬───────────┘    │
└─────────┼──────────────────────┼────────────────┘
          │                      │
          ▼                      ▼
┌──────────────────────────────────────────────────┐
│           Mock API 서버 (테스트 케이스)             │
│  - 복약 관리 계획 API                              │
│  - 방문 기록 API                                   │
│  - 의약품 교환 API                                 │
│  - 시나리오별 고정 응답 반환                        │
└──────────────────────────────────────────────────┘
```

### 6.4 데이터 모델 (주요 엔티티)

| 엔티티 | 주요 속성 |
|---|---|
| 환자 | 환자ID, 이름, 연락처, 주요 질환 |
| 처방 | 처방ID, 환자ID, 처방 의약품 목록, 총 처방 기간 |
| 복약 관리 계획 | 계획ID, 처방ID, 방문 횟수, 조제 단위, 방문 일정 목록 |
| 방문 기록 | 방문ID, 계획ID, 방문일, 복약 상태, 이상 반응, 조제 수량 |
| 의약품 교환 이력 | 교환ID, 방문ID, 교환 사유, 교환 수량 |

## 8. 성공 지표 (KPI)

### 8.1 환자 안전 지표
- 의약품 부작용 발생률 감소 (기준 대비 %)
- 유통기한 만료 의약품 복용 사고 건수: 0건 목표
- 복약 순응도 향상율

### 8.2 약국 운영 지표
- 장기 처방 환자 1인당 평균 방문 횟수 증가
- 복약 관리 서비스 등록 환자 수
- 고객 만족도 (NPS)

### 8.3 비즈니스 지표
- 복약 관리 관련 부가 서비스 매출
- 신규 고객 유입율 (관리 환자 소개 통한)
- 환자 재방문율

## 9. 제약 사항 및 가정

### 9.1 제약 사항
- 기존 약국 처방 조제 시스템(DUR 등)과의 연동 필요
- 의약품 분할 조제에 관한 법적 요건 검토 필요
- 약사 1인당 관리 가능한 환자 수 현실적 한계 고려

### 9.2 가정
- 장기 처방(30일 이상) 환자를 주 대상으로 함
- 환자가 스마트폰 사용 가능 (알림 기능 활용)
- 약국이 기본적인 IT 인프라(인터넷, PC 등) 보유
