# 구현 계획 작성 스킬

## 계획서 구조

### 1. 컨텍스트 (Context)
- 현재 상태
- 구현 목표
- 관련 기능 ID

### 2. 파일 변경 목록
- 수정할 파일 (변경 내용 요약)
- 신규 생성 파일
- 삭제할 파일

### 3. 구현 순서
- 백엔드 먼저 (API → 데이터)
- 프론트엔드 다음 (타입 → 컴포넌트 → 페이지)
- 테스트 마지막

### 4. 완료 기준
- 기능 동작 확인 항목
- 에러 케이스 처리 확인
- 반응형 레이아웃 확인

## 사용 방법
계획 수립 시 이 스킬을 호출하여 구현 전 전체 범위와 순서를 정리한다.
