# Karpathy 개발 가이드라인

## 핵심 원칙

### 1. 단순하게 시작하라
- 가장 단순한 구현으로 시작
- 복잡성은 필요할 때만 추가
- 과도한 추상화 지양

### 2. 점진적으로 구현하라
- 작은 단위로 구현하고 즉시 검증
- 한 번에 너무 많은 것을 바꾸지 않기
- 각 단계에서 동작을 확인

### 3. 코드를 읽기 쉽게 작성하라
- 명확한 변수명과 함수명
- 복잡한 로직에는 한국어 주석
- 짧은 함수와 컴포넌트

### 4. 테스트 가능하게 설계하라
- 순수 함수로 비즈니스 로직 분리
- 의존성 주입으로 테스트 용이성 확보

## 적용 방법
- 구현 전: 가장 단순한 방법을 먼저 시도
- 구현 중: 200줄 이상의 파일은 분리 검토
- 구현 후: 테스트 없는 비즈니스 로직은 테스트 추가
