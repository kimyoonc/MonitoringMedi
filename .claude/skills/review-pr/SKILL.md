# PR 코드 리뷰 스킬

## 리뷰 단계

1. 변경된 파일 목록 파악 (`git diff main...HEAD`)
2. 각 파일을 순서대로 읽고 이슈 수집
3. 이슈를 3단계로 분류하여 보고서 작성

## 이슈 분류 기준

### 🔴 Critical (반드시 수정)
- 보안 취약점 (XSS, SQL Injection, 민감 정보 노출)
- 런타임 에러 유발 가능 코드
- 데이터 유실·무결성 오류
- 빌드 실패 원인

### 🟡 Important (수정 권장)
- 성능 저하 (불필요한 리렌더링, N+1 쿼리 등)
- 에러 핸들링 누락
- 타입 안전성 문제
- 접근성(a11y) 이슈

### 🟢 Suggestion (선택적 개선)
- 코드 가독성 개선
- 중복 제거·리팩터링 제안
- 네이밍 개선
- 테스트 추가 제안

## 보고서 형식

```
## 코드 리뷰 결과

### 🔴 Critical (N건)
- [파일명:줄번호] 문제 설명 → 개선 방안

### 🟡 Important (N건)
- [파일명:줄번호] 문제 설명 → 개선 방안

### 🟢 Suggestion (N건)
- [파일명:줄번호] 문제 설명 → 개선 방안

### ✅ 잘된 점
- 긍정적인 부분 언급
```
