# Debug 스킬

## 실행 절차

### 1. 증상 파악
- 에러 메시지 전문 확인 (스택 트레이스 포함)
- 재현 조건 파악 (항상 발생 / 특정 조건에서만)
- 최근 변경사항 확인 (`git log --oneline -10`)

### 2. 원인 분석
- 에러 발생 위치 코드 읽기
- 관련 파일·함수 추적
- 가설 수립 (1~3개)

### 3. 수정
- 가장 가능성 높은 가설부터 검증
- 최소한의 변경으로 수정
- 사이드 이펙트 확인

### 4. 검증
- 빌드 성공 확인
- 에러 재현 시도 → 미발생 확인
- 관련 기능 정상 동작 확인

## 디버깅 원칙

- 브루트 포스(무작위 수정) 금지 — 반드시 원인을 이해한 후 수정
- 한 번에 하나씩만 변경하여 원인 격리
- 동일 에러 반복 시 다른 접근법 시도
- 수정 후 `npm run build`로 빌드 오류 없는지 확인

## 보고 형식

```
## 디버그 결과

**증상**: 에러 메시지 또는 증상 요약
**원인**: 파악된 근본 원인
**수정 내용**: 변경한 파일·코드 설명
**검증**: 수정 후 확인한 내용
```
